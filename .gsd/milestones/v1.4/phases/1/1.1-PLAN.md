# Plan 1.1: GEO/SEO Technical Audit

## Objective
Analyze the current codebase to identify gaps in SEO, technical structure, and semantic data. This will establish the baseline "AI Visibility Score" and define the specific work packages for Phase 2 and 3.

## Context
- .gsd/SPEC.md
- .gsd/ROADMAP.md
- app/layout.tsx
- app/robots.ts
- app/sitemap.ts

## Tasks

<task type="auto">
  <name>Audit Technical Foundation</name>
  <files>app/robots.ts, app/sitemap.ts, app/layout.tsx</files>
  <action>
    Analyze the current configuration for:
    - `robots.txt` logic (is it correctly handling the `IS_DOCKER_BUILD` flag?)
    - `sitemap.ts` generation (does it cover dynamic events? is the Content-Type correct?)
    - `layout.tsx` metadata (templates, default tags)
    Create a finding report in `.gsd/audit/finding_01_tech.md`.
  </action>
  <verify>Test-Path .gsd/audit/finding_01_tech.md</verify>
  <done>Clear understanding of technical gaps.</done>
</task>

<task type="auto">
  <name>Audit Semantic Coverage</name>
  <files>app/e/[slug]/page.tsx, components/StructuredData.tsx</files>
  <action>
    Scan key public pages (Landing, Event Page, Guide) for JSON-LD Schema.
    - Identify missing schemas (Event, FAQ, Guide, Organization).
    - Check for hardcoded strings vs typed objects.
    Create a finding report in `.gsd/audit/finding_02_schema.md`.
  </action>
  <verify>Test-Path .gsd/audit/finding_02_schema.md</verify>
  <done>Gap analysis of Schema.org implementation.</done>
</task>

<task type="auto">
  <name>Audit Content & Entities</name>
  <files>.gsd/audit/finding_03_content.md</files>
  <action>
    Analyze the current content against high-value entities (TTRPG, D&D, Scheduling).
    - Create a list of "Target Entities" (e.g., "Game Master", "Session", "Availability").
    - Identify gaps in current copy (e.g., Landing page missing "D&D Scheduler" explicit terms).
    - Draft a "Content Vocabulary" for future blog posts.
    Save findings to `.gsd/audit/finding_03_content.md`.
  </action>
  <verify>Test-Path .gsd/audit/finding_03_content.md</verify>
  <done>Vocabulary list and content gap report created.</done>
</task>

<task type="auto">
  <name>Compile Visibility Report</name>
  <files>.gsd/audit/REPORT.md</files>
  <action>
    Synthesize findings into a comprehensive report.
    - Calculate a heuristic "AI Visibility Score" (0-100).
    - Prioritize fixes for Phase 2.
    - Establish the "Target State" for each gap.
  </action>
  <verify>Test-Path .gsd/audit/REPORT.md</verify>
  <done>A finalized audit report for user review.</done>
</task>

## Success Criteria
- [ ] Technical gaps (sitemap/robots) documented.
- [ ] Schema coverage mapped.
- [ ] Content vocabulary defined.
- [ ] Actionable remediation list created for Phase 2.
